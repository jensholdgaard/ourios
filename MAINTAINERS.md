Ourios currently has a single maintainer. There is no formal
governance body; decisions are made by the maintainer through the
RFC process (`docs/rfcs/README.md`) — design changes are proposed,
discussed, and accepted as RFCs in the open, so the reasoning behind
every architectural decision is on the record even with one person
holding the pen.

Development is intentionally and openly AI-assisted: much of the
code is written by Claude under the maintainer's direction, and goes
through the same review, CI, and RFC gates as any contribution
(`CONTRIBUTING.md`).

## Becoming a maintainer

Pre-release, there is no formal path yet. Sustained, high-quality
contributions through the RFC and review process are how trust gets
built; this section will grow a real process when the contributor
base does.
